# TASK-006: Legat — Наименования компании (names)

**Дата создания:** 2026-04-06 00:00 (UTC+3, Брест)  
**Статус:** Завершена  
**Приоритет:** Высокий  
**Исполнитель:** Bitrix24 Программист (Vanilla JS)

## Описание
Реализовано кастомное activity `legatcompanynames`: получение сведений о наименованиях компании из ЕГР по УНП через `GET /api2/by/names`.

## Контекст
Activity дополняет линейку методов Legat (`data`, `contacts`, `court`, `okved`, `address`) и возвращает историю/актуальность наименований компании для использования в бизнес-процессах Битрикс24.

Ключ API передается:
- в GET-параметре `key`, либо
- в HTTP-заголовке `key`.

В реализации проекта ключ должен оставаться на сервере и не отображаться в UI.

## Модули и компоненты
- `local/activities/custom/legatcompanynames/` — activity
- `local/activities/custom/LegatByClient.php` — общий HTTP-клиент и работа с ключом API

## API
- Метод: `GET /api2/by/names`
- URL: `https://api.legat.by/api2/by/names`
- Параметры:
  - `key` (string, required) — ключ доступа
  - `unp` (string, required) — учетный номер (УНП)

Пример запроса:
`https://api.legat.by/api2/by/names?unp=692172505&key=***`

## Набор возвращаемых полей
По каждой записи `names[]`:
- `unp` — УНП субъекта хозяйствования
- `full` — полное наименование субъекта хозяйствования
- `date` — дата внесения сведений в ЕГР о присвоении наименования
- `date_end` — дата внесения сведений в ЕГР о прекращении использования наименования
- `active` — признак активности (`1` — действующее, `0` — историческое)

Верхний уровень ответа:
- `error`
- `total`
- `names`

## Пример ответа сервера
```json
{
  "error": null,
  "total": 1,
  "names": [
    {
      "unp": "692172505",
      "full": "Общество с ограниченной ответственностью \"Лифтман\"",
      "date": "2020-11-13",
      "date_end": "0000-00-00",
      "active": 1
    }
  ]
}
```
