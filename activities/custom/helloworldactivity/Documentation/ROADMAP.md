## 5. Заметки и идеи

### 5.1 Архитектурные решения

- **Единый базовый класс:** Все действия наследуют от `BaseActivity` (D7)
- **Свойства через `arProperties`:** Стандартный подход, а не прямые поля
- **Типы через `SetPropertiesTypes`:** Явное определение типов
- **Возврат через `preparedProperties`:** Результат сохраняется здесь

### 5.2 Полезные паттерны

```php
// Паттерн: Валидация параметров
protected function internalExecute(): ErrorCollection
{
    $errors = parent::internalExecute();
    
    if (empty(trim($this->SomeParam))) {
        $errors[] = new \Bitrix\Main\Error('Параметр не может быть пустым', 'EMPTY_PARAM');
        return $errors;  // Ранний возврат при ошибке
    }
    
    // Основная логика...
    return $errors;
}

// Паттерн: Работа с CRM
use Bitrix\Crm\DealTable;

$deal = DealTable::getById($this->EntityId)->fetch();
if (!$deal) {
    $errors[] = new \Bitrix\Main\Error('Сделка не найдена', 'DEAL_NOT_FOUND');
}

// Паттерн: HTTP-запрос
use Bitrix\Main\Web\HttpClient;

$httpClient = new HttpClient();
$response = $httpClient->get($this->Url);
```
