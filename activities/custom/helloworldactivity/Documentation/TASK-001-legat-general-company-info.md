# TASK-001: Legat- Общие сведения о компании

**Дата создания:** 2026-04-06 00:00 (UTC+3, Брест)
**Статус:** Завершена
**Приоритет:** Высокий
**Исполнитель:** Bitrix24 Программист (Vanilla JS)

## Описание
Создать новое кастомное activity для бизнес-процессов Битрикс24, которое получает общие сведения о компании из API Legat по УНП, подставляемому из CRM глобально, и возвращает результат запроса в выходное поле `Доп сведения` для дальнейшего использования в бизнес-процессе.

## Контекст
Действие нужно для быстрого получения сведений о компании из внешнего сервиса Legat прямо в бизнес-процессе Битрикс24. Токен доступа не показывается в UI и подгружается из отдельного файла `local/legat_api_key.php` (с fallback на `Option main.legat_by_data_api_key`). Результат ответа раскладывается по структуре полей в дополнительных результатах выполнения activity.

## Модули и компоненты
- `local/activities/custom/legatcompanyinfo/.description.php` — метаданные activity
- `local/activities/custom/legatcompanyinfo/legatcompanyinfo.php` — основной PHP-класс activity
- `local/activities/custom/legatcompanyinfo/lang/ru/.description.php` — локализация описания
- `local/activities/custom/legatcompanyinfo/lang/ru/legatcompanyinfo.php` — локализация полей и сообщений
- `local/activities/custom/legatcompanyinfo/README.md` — краткая инструкция по установке и использованию
- `local/legat_api_key.php` — отдельный файл с ключом API (define-константа)

## Зависимости
- Доступность API Legat `GET /api2/by/data`
- Наличие УНП в данных CRM, откуда его можно подставить глобально
- Наличие ключа доступа в `local/legat_api_key.php` или в `Option main.legat_by_data_api_key`
- Действие должно работать внутри механизма кастомных activity Битрикс24

## API-методы
- `GET /api2/by/data`
- Документация: `https://api.legat.by/api2/by/data`
- Параметры запроса:
  - `key` — ключ доступа
  - `unp` — учетный номер плательщика

## Технические требования
- Платформа: Bitrix24
- Тип фронтенда: Vanilla JS
- Токен доступа Legat хранить внутри кода activity
- УНП брать из CRM-глобального значения, как в текущем activity берется `Объект`

- Обязательно обрабатывать ошибки сети и ошибки API
