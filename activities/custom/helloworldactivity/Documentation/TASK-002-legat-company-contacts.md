## Описание
Реализовать новое кастомное activity для бизнес-процессов Битрикс24, которое получает сведения о контактах компании из API Legat по УНП через метод `GET /api2/by/contacts` и возвращает данные в дополнительные выходные поля activity для использования в следующих шагах БП.

## Контекст
В проекте уже есть рабочая реализация activity для метода `GET /api2/by/data` (общие сведения о компании). Нужно сделать аналогичный рабочий подход для метода контактов:
- телефоны компании;
- факс;
- email;
- сайты.

Ключ API должен оставаться скрытым от интерфейса пользователя и браться из серверной конфигурации:
- `local/legat_api_key.php` (приоритет),
- fallback: `Option::get('main', 'legat_by_data_api_key')`.

## API-методы (Legat)
- `GET /api2/by/contacts`
- Полный URL: `https://api.legat.by/api2/by/contacts`
- Параметры запроса:
  - `key` (string, required) — ключ доступа;
  - `unp` (string, required) — учетный номер (УНП).

Пример запроса:
`https://api.legat.by/api2/by/contacts?unp=692172505&key=***`

Пример ответа:
```json
{
  "error": null,
  "contacts": {
    "phone": [
      "+375293956956",
      "+375293956938",
      "+375293956987"
    ],
    "fax": null,
    "email": [
      "info@liftmann.by",
      "buh@liftmann.by",
      "tender@liftmann.by"
    ],
    "site": [
      "liftmann.by"
    ]
  }
}
```

## Технические требования
- Платформа: Bitrix24 (кастомные activity).
- Реализация: PHP + стандартные механизмы Bitrix24, без внешних JS-фреймворков.
- Ключ API не выводить в UI и не хранить в репозитории.
- Выходные данные должны быть доступны как отдельные поля activity.
- Для множественных значений (телефоны/email/сайты) предусмотреть стабильный формат:
  - либо массивы в `preparedProperties`,
  - либо строка через разделитель (например, `; `) с документированным форматом.
- Непустой `REQUEST_STATUS` обязателен в каждом сценарии (успех/ошибка).

## Предлагаемая структура выходных полей
- `REQUEST_STATUS` — строка статуса выполнения (`Успешно` / текст ошибки)
- `CONTACTS_PHONE` — телефоны компании
- `CONTACTS_FAX` — факс
- `CONTACTS_EMAIL` — email компании
- `CONTACTS_SITE` — сайты компании
- `CONTACTS_RAW_JSON` — опционально: сырой JSON для отладки и расширения
