## 1. Общая архитектура бизнес-процессов

### 1.1 Что такое модуль «Бизнес-процессы»

Модуль `bizproc` — это ядро автоматизации Битрикс24. Он отвечает за:
- Создание и исполнение бизнес-процессов
- Управление шаблонами БП
- Выполнение действий (activities)
- Работу с переменными и параметрами БП
- Обработку событий и состояний

### 1.2 Основные компоненты системы

```
┌─────────────────────────────────────────────────────────┐
│                    Бизнес-процесс                        │
│                                                         │
│  ┌───────────┐   ┌───────────┐   ┌───────────┐         │
│  │ Action 1  │──▶│ Action 2  │──▶│ Action 3  │         │
│  └───────────┘   └───────────┘   └───────────┘         │
│       │               │               │                 │
│       ▼               ▼               ▼                 │
│  ┌───────────┐   ┌───────────┐   ┌───────────┐         │
│  │ Properties│   │ Properties│   │ Properties│         │
│  │  (вход)   │   │  (вход)   │   │  (выход)  │         │
│  └───────────┘   └───────────┘   └───────────┘         │
└─────────────────────────────────────────────────────────┘
```

### 1.3 Типы действий

| Тип | Описание | Пример |
|-----|----------|--------|
| **Системные** | Встроенные действия Битрикс | Отправка письма, создание задачи |
| **Кастомные** | Разработанные разработчиком | Логирование, расчёты, интеграции |
| **Группы** | Контейнеры для других действий | Параллельные/последовательные группы |

## 2. Жизненный цикл бизнес-процесса

### 2.1 Этапы выполнения

```
1. ИНИЦИАЛИЗАЦИЯ
   ├── Загрузка шаблона БП
   ├── Создание экземпляра процесса
   └── Установка переменных и параметров

2. ВЫПОЛНЕНИЕ
   ├── Последовательное выполнение действий
   ├── Обработка условий и переходов
   └── Сохранение промежуточных состояний

3. ЗАВЕРШЕНИЕ
   ├── Фиксация результатов
   ├── Очистка ресурсов
   └── Генерация событий завершения
```

### 2.2 Состояния действия

```
CREATED → INITIALIZED → EXECUTING → COMPLETED
                                     │
                                     ▼
                                ERROR (если ошибка)
```

## 3. Архитектура кастомного действия

### 3.1 Наследование классов

```
Bitrix\Bizproc\Activity\BaseActivity  ← базовый класс D7
              ↑
    CBPActivity                  ← старый базовый класс
              ↑
    CBPMyCustomActivity          ← ваше действие
```

### 3.2 Ключевые методы BaseActivity

| Метод | Назначение |
|-------|-----------|
| `__construct($name)` | Инициализация свойства и типов |
| `internalExecute()` | **Основная логика** действия |
| `getPropertiesDialogMap()` | Конфигурация диалога настроек |
| `getFileName()` | Путь к файлу действия |
| `initialize()` | Предварительная инициализация |
| `execute()` | Обёртка над internalExecute() |

### 3.3 Поток данных внутри действия

```
┌──────────────────────────────────────────────────────┐
│                  Action Flow                         │
│                                                      │
│  ┌─────────────┐    ┌──────────────┐    ┌─────────┐ │
│  │  ВХОДНЫЕ    │───▶│ internal-    │───▶│  ВОЗВРА- │ │
│  │  ПАРАМЕТРЫ  │    │  Execute()   │    │  ЩАЕМЫЕ │ │
│  │  (properties)│  │  (логика)    │    │  VALUES │ │
│  └─────────────┘    └──────────────┘    └─────────┘ │
│                                                      │
│  Subject ──────────┐                                 │

│                    │                                 │

```

## 4. Система типов полей (FieldType)

### 4.1 Полный список типов

```php
use Bitrix\Bizproc\FieldType;

FieldType::STRING      // Однострочный текст
FieldType::TEXT        // Многострочный текст
FieldType::INT         // Целое число
FieldType::DOUBLE      // Дробное число
FieldType::BOOL        // Булево значение
FieldType::SELECT      // Выпадающий список
FieldType::MULTIPLE    // Множественный выбор
FieldType::USER        // Пользователь
FieldType::DOCUMENT    // Документ
FieldType::DATE        // Дата
FieldType::DATETIME    // Дата и время
FieldType::MAILER      // Почтовый шаблон
```

### 4.2 Структура описания поля

```php
'PropertyName' => [
    'Name'        => 'Отображаемое имя',      // Название поля в диалоге
    'FieldName'   => 'property_name',          // HTML-имя поля
    'Type'        => FieldType::STRING,        // Тип поля
    'Required'    => true,                     // Обязательно ли
    'Default'     => 'значение',               // Значение по умолчанию
    'Options'     => [                         // Опции (для SELECT)
        ['value1' => 'Вариант 1'],
        ['value2' => 'Вариант 2'],
    ],
    'Description' => 'Подсказка',              // Описание поля
],
```

## 5. Возвращаемые значения

### 5.1 Как это работает

Действие может возвращать значения, которые становятся доступными для:
- Использования в последующих действиях
- Условий и переходов
- Шаблонов документов и писем

### 5.2 Типы возвращаемых значений

```php
"RETURN" => [
    "VariableName" => [
        "NAME" => "Имя переменной",
        "TYPE" => "string",    // string, int, double, bool, date, user
    ],
],
```

### 5.3 Пример использования

```php
// В .description.php
"RETURN" => [
    "CalculatedSum" => [
        "NAME" => "Итоговая сумма",
        "TYPE" => "double",
    ],
],

// В классе действия
protected function internalExecute(): ErrorCollection
{
    $sum = $this->Price * $this->Quantity;
    $this->preparedProperties['CalculatedSum'] = $sum;
    return $errors;
}
```

## 6. Обработка ошибок

### 6.1 ErrorCollection

Все ошибки возвращаются через `Bitrix\Main\ErrorCollection`:

```php
use Bitrix\Main\ErrorCollection;
use Bitrix\Main\Error;

protected function internalExecute(): ErrorCollection
{
    $errors = parent::internalExecute();
    
    if ($this->Subject === '') {
        $errors[] = new Error('Поле "Объект" обязательно', 'EMPTY_SUBJECT');
    }
    
    return $errors;
}
```

### 6.2 Типы ошибок

| Тип | Описание |
|-----|----------|
| **Критические** | Остановка выполнения БП |
| **Некритические** | Предупреждение, продолжение выполнения |

## 10. Best Practices

### 10.1 ✅ Хорошие практики

- Называйте классы с префиксом `CBP`
- Всегда вызывайте `parent::internalExecute()`
- Используйте локализацию для всех строк
- Документируйте возвращаемые значения
- Обрабатывайте ошибки явно

### 10.2 ❌ Чего избегать

- Жёстко закодированных строк
- Глобальных переменных
- Прямых SQL-запросов без необходимости
- Игнорирования ошибок
- Сложной бизнес-логики в одном действии
